# Block 8.6 — Library + Transcript Detail UI + Production Composition Root

**Date:** 2026-05-28
**Branch:** `feat/block-08-android-skeleton`
**Status:** Approved design, ready for implementation plan

## Context & goal

Block 8 has a complete, JVM-tested data + protocol + view-model stack (8.1–8.5), but
every Compose screen is a 22-line placeholder and `App.kt` is an empty `Application`
with no dependency graph. The view-models (`LibraryViewModel`,
`TranscriptDetailViewModel`) are plain classes that take a `CoroutineScope` and their
collaborators by constructor; nothing wires them to a real Room database, OkHttp client,
crypto stack, or the Android lifecycle.

This block delivers the **Library list screen**, the **Transcript detail screen**, the
**navigation** between them, and the **full production composition root** that the
handoff listed as a separate follow-up. After this block the app launches into a real,
data-backed Library screen (empty until a hub is paired) and can open a transcript into
a rendered detail view.

Goal slice of the block-8.0 verification target ("list transcripts, open one, render
correctly") is satisfied at the UI + wiring level. The remaining half (actually
populating the library) depends on block 8.3b (pairing UI) and two hub-side gaps, which
are explicitly **out of scope** here — see "Consequences."

## Locked decisions

| # | Decision |

## Architecture

### Composition root — `AppGraph`

A single lazily-built, application-scoped graph. One `EncryptedSharedPreferences` file
backs all four prefs-backed stores (keys do not collide), one `LazySodiumAndroid`, one
`OkHttpClient`, one `LocalLexisDatabase`.

```
class AppGraph(context: Context) {
    private val securePrefs: SharedPreferences        // single EncryptedSharedPreferences file "locallexis_secure"
    val sodium: LazySodium                            // LazySodiumAndroid(SodiumAndroid())
    val secretStorage: SecretStorage                  // EncryptedPrefsSecretStorage(securePrefs)   [NEW]
    val cryptoBox: CryptoBox                           // LazysodiumCryptoBox(secretStorage, sodium)
    val workspaceKeyStore: WorkspaceKeyStore          // EncryptedPrefsWorkspaceKeyStore(securePrefs)
    val deviceIdentityStore: DeviceIdentityStore      // PrefsDeviceIdentityStore(securePrefs)
    val hubConfig: HubConfigStore                     // PrefsHubConfigStore(securePrefs)            [NEW]
    val db: LocalLexisDatabase                        // LocalLexisDatabase.get(context)
    val okHttp: OkHttpClient                          // .addInterceptor(SignedRequestInterceptor(cryptoBox, deviceIdentityStore))
    val pairingClient: PairingClient                  // (cryptoBox, workspaceKeyStore, deviceIdentityStore, okHttp)

    fun librarySync(): LibrarySync                    // real DefaultLibrarySync when paired, else UnpairedLibrarySync
    fun workspaceId(): String                         // hubConfig.getWorkspaceId() ?: ""
}
```

- `securePrefs` is built once via the same `MasterKey` + `EncryptedSharedPreferences.create`
  pattern already in `EncryptedPrefsWorkspaceKeyStore.create`, but here the prefs object
  is shared and passed into all four store constructors (each store exposes a
  `SharedPreferences`-taking constructor, so the `create()` helper is bypassed).
- Built lazily on first access. Prefs/Keystore init is cheap but touches the Android
  Keystore; the graph is created off the hot path (first screen composition), acceptable
  for this block. Flagged as a risk below.
- Held by `App`: `class App : Application() { val graph by lazy { AppGraph(this) } }`.
  Screens reach it via `(LocalContext.current.applicationContext as App).graph`.

### `librarySync()` — paired vs unpaired

`LibraryViewModel` requires a non-null `LibrarySync` and a `workspaceId`. When no hub is
paired (`hubConfig` returns null), there is no `baseUrl` to build a `SyncClient`.

```
fun librarySync(): LibrarySync {
    val base = hubConfig.getHubUrl() ?: return UnpairedLibrarySync
    return DefaultLibrarySync(SyncClient(okHttp, base), SyncIngest(db))
}

object UnpairedLibrarySync : LibrarySync {            // [NEW]
    override suspend fun bootstrap() = throw NotPairedException()
    override suspend fun incremental(workspaceId: String) = throw NotPairedException()
}
class NotPairedException : RuntimeException("no hub paired — pair a hub to sync")
```

`LibraryViewModel.refresh()` already catches `Throwable` into `lastError` without
clearing the row list, so an unpaired `refresh()` shows "no hub paired — pair a hub to
sync" in the error banner and leaves the (empty) list intact. No crash, no special-casing
in the VM.

### hub_url / workspace_id persistence fix

`SyncClient` needs `baseUrl = hub_url`; `LibraryViewModel` needs `workspaceId`. Today
`PairingClient.exchange` persists only the workspace key `W` and `device_id`; it returns
`workspace_id` and silently drops `hub_url`. Nothing stores either value, so the graph
cannot build the sync stack from persisted state.

**Fix without touching `PairingClient` or its tests:** add a `HubConfigStore` and have the
*production pairing wiring* (the lambda passed to `PairingViewModel`, future block 8.3b)
write into it:

```
val pair: suspend (PairingPayloadV1, String) -> PairingResult = { payload, name ->
    pairingClient.exchange(payload, name).also { result ->
        hubConfig.put(hubUrl = payload.hubUrl, workspaceId = result.workspaceId)
    }
}
```

`PairingClient.kt` and `PairingClientTest.kt` stay untouched. This block adds the store
and the graph plumbing; 8.3b will consume the `pair` lambda. (The lambda is defined in
`AppGraph` so it is ready for 8.3b, but no UI calls it yet this block.)

### New stores

```
interface HubConfigStore {

class PrefsHubConfigStore(prefs: SharedPreferences) : HubConfigStore   // keys: "hub_url", "workspace_id"

// data/crypto/EncryptedPrefsSecretStorage.kt          [NEW]

```

Shared-prefs key map (one file, no collisions): `workspace_key_b64`, `device_id`,
`device_seed_b64`, `hub_url`, `workspace_id`.

### Navigation

`MainScreen` keeps its 4-tab bottom bar. Add one route to the same `NavHost`:

```
composable("transcript/{transcriptId}") { entry ->
    val id = entry.arguments?.getString("transcriptId").orEmpty()
    TranscriptDetailScreen(transcriptId = id)
}
```

`LibraryScreen` gains an `onOpen: (String) -> Unit` callback; `MainScreen` passes
`{ id -> nav.navigate("transcript/$id") }`. Bottom bar remains visible on the detail
route (minimal-change decision 1b). Back navigation uses the system/`NavController` back
stack.

### ViewModel bridge

Thin `ViewModel` wrappers create the plain VMs with `viewModelScope`:

```
class LibraryViewModelHolder(graph: AppGraph) : ViewModel() {
    val vm = LibraryViewModel(graph.db.transcriptDao(), graph.librarySync(), graph.workspaceId(), viewModelScope)
}
class TranscriptDetailViewModelHolder(graph: AppGraph, id: String) : ViewModel() {
    val vm = TranscriptDetailViewModel(graph.db, id, viewModelScope)
}
```

Each holder has a companion `factory(...)` returning a `ViewModelProvider.Factory`
(`viewModelFactory { initializer { … } }`). Detail's factory closes over the
`transcriptId` route argument. The wrappers contain no logic beyond construction, so they
need no tests; all behavior remains in the already-tested plain VMs.

## Screens

Each screen splits into a **stateless content** composable (driven purely by state +
callbacks, the `@Preview` target) and a **stateful wrapper** (collects the VM).

### LibraryScreen (layout B — ledger rows)

- `LibraryScreen(onOpen)` → resolves `AppGraph`, builds `LibraryViewModelHolder` via
  `viewModel(factory)`, collects `vm.uiState` + `vm.lastError`, renders `LibraryContent`.
- `LibraryContent(state, lastError, onOpen, onRefresh)`:
  - **Loading** → centered `CircularProgressIndicator`.
  - **Ready(empty)** → empty state: "No transcripts yet" + secondary "Pair a hub to sync
    your library." (matches the unpaired reality).
  - **Ready(list)** → `LazyColumn` of ledger rows + a refresh affordance (top app-bar
    action or pull-to-refresh; app-bar action chosen to avoid an extra dependency).
  - **Error(message)** → centered message + Retry button (`onRefresh`).
  - `lastError != null` → dismissible banner at top; the row list stays visible beneath.
  - **Row**: `Row { Column(weight=1f) { Text(title, titleMedium); Text("$date · $lang", bodySmall, muted) }; Text(duration, accent, tabular) }`, full-row `clickable { onOpen(id) }`, `HorizontalDivider` between rows. `title = audioBasename ?: id`.

### TranscriptDetailScreen (layout B — speaker bubbles)

- `TranscriptDetailScreen(transcriptId)` → builds `TranscriptDetailViewModelHolder` via
  `viewModel(factory)`, collects `vm.uiState`, renders `TranscriptDetailContent`.
- `TranscriptDetailContent(state)`:
  - **Loading** → spinner.
  - **NotFound** → "Transcript not found."
  - **Error(message)** → message.
  - **Ready(transcript, segments)** → `Scaffold`/`Column`:
    - Header: title (`headlineSmall`), meta line `date · duration · lang · "N speakers"`
      (N = distinct non-blank `speakerName`s).
    - `LazyColumn` of bubbles: `Surface(color = speakerHue(name), shape = rounded) {
      Column { Text("$name · $mmss", labelSmall, accent); Text(text, bodyMedium) } }`.
      `mmss` from segment `startSec`.

### Formatting + speaker color helpers (`ui/format/`)

```
// Formatting.kt
fun formatDuration(seconds: Double?): String   // null/blank → ""; "m:ss" or "h:mm:ss"
fun formatDate(iso: String?): String           // ISO-8601 → "MMM d, yyyy" (Locale.getDefault); fallback raw/"" on parse failure

// SpeakerColors.kt
fun speakerHue(name: String): Color            // deterministic index = floorMod(name.hashCode(), PALETTE.size); ~6 parchment-safe tints w/ enough contrast for ink text
```

`formatDate` uses `java.time` (minSdk 26, safe). Parse failure is non-fatal: return the
raw string (or "" when null) so an unexpected `createdAt` format never crashes the row.

## Consequences (chosen "full graph, no seed" path)

- The **running app shows an empty Library** ("pair a hub to sync") until block 8.3b +
  the two hub-side gaps land. Populated list + bubble rendering is verified via `@Preview`
  this block, not in the live app.
- The production `pair` lambda (writing `HubConfigStore`) is defined in `AppGraph` and
  ready, but unused until 8.3b wires `PairingViewModel` to the camera/QR flow.

## Out of scope (unchanged follow-ups)

- 8.3b pairing UI (CameraX + ML Kit + permission flow).
- Hub gap 1 (`/sync/snapshot` missing per-transcript `id`) and gap 2 (hub UI emits
  `MintTokenResponse`, not `PairingPayloadV1`).
- OkHttp `CertificatePinner` wiring (`PairingPayloadV1.tlsSpkiB64`).
- `SettingsScreen` content (workspace id, paired-device count, "reset pairing" button) —
  natural next screen, but not required for the list/open/render goal.

## Risks

- **`EncryptedSharedPreferences` init** touches the Android Keystore; first-access cost on
  the composition thread. Acceptable for this block; revisit with an off-main warm-up if
  it shows in profiling.
- **`createdAt` wire format** is assumed ISO-8601; `formatDate` falls back to raw on any
  parse failure, so a format surprise degrades to a readable raw string, never a crash.
- **`speakerHue` collisions** — distinct speakers may share a tint via hash modulo; purely
  cosmetic, names are always shown in the bubble header.
